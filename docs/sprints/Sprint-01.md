# Sprint 1 - Project Foundation

## Sprint Goal

Create the foundation for the Verdis project by setting up the development environment, project structure, documentation, and version control.

---

## User Stories

- [x] Choose project name
- [x] Create GitHub repository
- [x] Configure Git and SSH
- [x] Install and configure Flutter
- [x] Verify emulator and application startup
- [x] Create initial project architecture
- [x] Create documentation structure

---

## Deliverables

- Flutter project created
- GitHub repository online
- Initial documentation
- Feature-first architecture
- Development environment ready

---

## Notes

This sprint focuses on preparing the project for development. No application features are implemented yet.

---


## Retrospective

### What went well

- Development environment successfully configured.
- GitHub repository created.
- Project architecture established.

### Challenges

- GitHub SSH authentication required additional configuration.
- Android emulator needed to be reconfigured.

### Lessons Learned

- GitHub no longer supports password authentication.
- Git only tracks files, not empty folders.
- A well-organized project structure is easier to maintain from the beginning.
